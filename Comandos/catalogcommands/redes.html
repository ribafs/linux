Comandos de Rede

Descrição

Nessa página será mostrado os principais comando de configuração de rede do Linux.

ifconfig

  • O ifconfig mostra os detalhes da(s) interface(s) de rede definida(s) no sistema.
    • -a: mostra todas as interfaces.
    • [nome da interface]: mostra os detalhes da interface especificada.

  • As partes importantes contidas nas informações de cada interface de rede são:
    • Link encap: o tipo de hardware físico compatível com esta interface.
    • HWaddr: Endereço MAC da placa de rede. O endereço MAC &e um identificador exclusivo atribu%iacutedo pelo fabricante. O identificador MAC não pode ser alterado.
    • Mask: a máscara de rede.
    • inet addr: o endereço IP anexado à interface.

  • Também é possível usar o ifconfig para administrar essas interfaces. Alguns dos comandos para administrar as interfaces podem ser vistos a seguir:
    • up/down – ativa ou desativa uma interface específica.
      ifconfig eth0 down
    • add – define um endereço IP específico para a interface.
      ifconfig eth0 add 192.168.1.101
    • netmask – define a máscara de rede da interface.
      ifconfig eth0 netmask 255.255.255.0

netstat

  • O comando netstat fornece as informações completas sobre o desempenho (estatísticas) da(s) interface(s) de rede.
    • -i: Mostra estatísticas das interfaces presentes.

      Saida:

      	Kernel  Interface table
	Iface   MTU Met   RX-OK RX-ERR RX-DRP RX-OVR  TX-OK TX-ERR TX-DRP TX-OVR Flg
	eth0   1500  0      844   0      0      0       551   0      0      0   BMRU
	lo    65536  0      365   0      0      0       365   0      0      0    LRU
    • -i -e: Mesma saí do comando ifconfig.
    • -s: Mostra um resumo estatístico de cada protocolo em vez de apresentar os detalhes de cada conexão.
    • -s -u/-t: Mostra as estatísticas relacionadas ao protocolo UDP/TCP.

      Saida:

   	88 packets to unknown port received.
   	0 packet receive errors
   	395 packets sent

ping

  • É utilizado para testar uma conexão, sendo que este utilitário de diagnóstico utiliza-se das mensagens Echo Request e Echo Reply do protocolo ICMP para determinar se uma máquina está ligada e funcional. Ele opera enviando um ICMP (Control Message Protocol), se o software de IP da máquina destino recbe-o ele emite uma resposta de echo imediatamente.
    ping [opções] google.com [IP ou hostname]

    Saida

        PING google.com (216.58.222.14) 56(84) bytes of data.
    64 bytes from gru06s25-in-f14.1e100.net (216.58.222.14): icmp_seq=1 ttl=55 time=202ms
    64 bytes from gru06s25-in-f14.1e100.net (216.58.222.14): icmp_seq=2 ttl=55 time=62.7ms
    64 bytes from from gru06s25-in-f14.1e100.net (216.58.222.14): icmp_seq=3 ttl=55 time=203ms
    64 bytes from from gru06s25-in-f14.1e100.net (216.58.222.14): icmp_seq=4 ttl=55 time=72.8ms
    64 bytes from from gru06s25-in-f14.1e100.net (216.58.222.14): icmp_seq=5 ttl=55 time=127ms
    ------google.com ping statistics ------
    5 packets transmitted, 5 received, 0% packets loss, time 4005ms
    rtt min/avg/max/mxdev = 62.767/133.799/203.342/60.547 ms

  • Algumas opções de comando para o ping são:
    • -c [x] : Faz o ping enviar x pacotes.
    • -i [n]: Aguarda n segundos para enviar o prõximo pacote
    • -R: Registra a rota seguida pelos pacotes at&eatilde chegar ao seu destino
    • -t [ttl]: Fixa o ttl (time to live), que indica o maior número de saltos que um pacote pode dar antes de ser descartado.

traceroute

  • Mostra o caminho percorrido por um pacote para chegar ao seu destino. Cada linha que aparece é chamada de pulo.
    Este comando mostra na tela o caminho percorrido entre os Gateways da rede e o tempo gasto de retransmissão.
    traceroute [opções] google.com [IP ou hostname]
