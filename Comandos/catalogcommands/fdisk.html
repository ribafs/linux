fdisk

fdisk [opções]

Descrição

Este comando manipula a tabela de partição do disco.

Algumas opções do comando

  • -b tamanho : especifica o tamanho do setor. São valores válidos: 512, 1024, 2048 or 4096. Usado apenas nas versões antigas do kernel.
  • -C tamanho : especifica o número de cilindros do disco.
  • -h : exibe informações sobre o fdisk.
  • -l : lista as partições existentes no disco atual (caso o usuário não especifique o disco).
  • -s partição : lista o tamanho da partição em blocos.
  • -v : exibe versão do aplicativo.

sudo fdisk -l

Inicialmente, é preciso entender que a configuração usada pelo sistema pode ser diferente da situação real do HD. Da saída acima, podemos tirar as seguintes conclusões sobre como o sistema ver o HD:

  • O disco tem 200 GBytes divididos em 390.721.968 setores, onde cada setor tem 512 Bytes .
  • Uma trilha tem 63 setores, portanto existem 390.721.968 / 63 = 6.201.936 trilhas.
  • Existem 255 cabeçotes (heads), logo o disco possui 255 superfícies e um cilindro possui 255 trilhas. Isto significa que em uma leitura/gravação são lidos 255 setores de uma vez, cada um em uma superfície diferente mas com um mesmo número de trilha.
Sobre a situação real do HD, podemos deduzir:
  • Normalmente, um HD de computador pessoal possui apenas um prato com dupla superfície. Para ler os dados, existem dois cabeçotes (heads): um cabeçote para cada superfície.
  • Neste exemplo, quando um cabeçote é posicionado, ele lê/grava 255 setores de uma vez.

Outra observação que pode ser feita sobre a saída do comando mostrada acima é que a primeira partição do HD começa no bloco 2.048 (2.048 X 512 Bytes = 1.048.576 Bytes). Isto significa que o primeiro MegaBytes do HD não é usado pelos sistemas de arquivos. Ele foi reservado para a instalação do MBR e do GRUB2.
