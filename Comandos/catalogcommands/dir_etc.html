/etc

Descrição

Este diretório possui os arquivos de configuração do sistema.

Exemplos de arquivos

  • adduser.conf - Este arquivo possui a configuração padrão dos comandos adduser e addgroup.
  • adjtime - Este arquivo possui o fator de correção para o RTC (real time clock, ou seja, relógio de tempo real).
  • at.allow - Este arquivo possui a lista dos usuários que podem usar o comando at. Se este arquivo não existir, todos os usuários têm permissão. Se este arquivo estiver vazio, apenas o root pode executar o comando at.
  • at.deny - Este arquivo possui a lista dos usuários que não podem usar o comando at. Se o arquivo /etc/at.allow não existir, é verificada a existência do arquivo at.deny e todos os usuários não mencionados no mesmo, recebem permissão para executar o comando at.
  • bash.bashrc - Este arquivo possui as funções e os aliases disponíveis a todos os usuários do sistema. As letras RC no nome bashrc significa Run Control File (arquivo de controle de execução).
  • crontab - Este é o arquivo crontab do sistema, ou seja, este é o arquivo que armazena as tarefas agendadas do sistema.
  • default - Este diretório armazena a configuração padrão de alguns comandos dos sistema.
  • fstab - Este arquivo mostra os dispositivos de hardware disponíveis no sistema (isto não significa que estejam montados) e é usado pelo swapon e mount durante o processo de inicialização do sistema.
  • group - Este arquivo possui as definições dos grupos do sistema. Existe uma entrada no arquivo para cada grupo de usuários do sistema com as seguintes informações: nome do grupo, senha do grupo (se houver), número de GID (group identification) e a lista dos usuário que pertencem a esse grupo. Por exemplo, a linha abaixo mostra a entrada criada em etc/group para o grupo sys que tem GID igual a 3 e que possui três membros: root, bin e adm.
    sys:x:3:root,bin,adm
  • gshadow - Este arquivo possui as senhas criptografadas dos grupos do sistema. Apenas o root (administrador do sistema) tem permissão para acessar o arquivo /etc/gshadow. O comando grpconv é usado para criar o arquivo gshadow a partir do arquivo /etc/group, enquanto o comando grpunconv executa a operação inversa.
  • host.conf - Este arquivo configura a ordem de procura dos nomes das máquinas.
  • hostname - Este arquivo possui o nome da máquina.
  • hosts - Este arquivo possui os endereços e os nomes das máquinas da rede.
  • hosts.allow - Este arquivo possui a lista das máquinas que possuem autorização para acessar os serviços de rede colocados disponíveis na máquina.
  • hosts.deny - Este arquivo possui a lista das máquinas que não podem acessar os serviços de rede colocados disponíveis na máquina.
  • init - Este diretório contém os arquivos de configuração inicializados pelo processo init.
  • init.d - Este diretório contém os scripts usados na inicialização do sistema nos diferentes níveis de execução (runlevels).
  • issue - Este arquivo é usado como tela de inicialização, no modo texto, dos usuários do sistema. Normalmente, este arquivo mostra o nome e a versão do sistema operacional.
  • issue.net - Este arquivo é usado como tela de inicialização em um login remoto (por exemplo, telnet).
  • ld.so.cache - Este arquivo possui as informações sobre as bibliotecas compartilhadas do sistema.
  • ld.so.conf - Este arquivo possui a lista dos diretórios onde são armazenadas as bibliotecas compartilhadas.
  • login.defs - Este arquivo possui a configuração padrão utilizada pelo comando useradd na criação de novas contas de usuários.
  • manpath.config - Este arquivo possui a lista dos diretórios onde as páginas do manual on-line (comando man) devem ser procuradas, além de definir a ordem em que a pesquisa por uma página do manual é feita.
  • mke2fs.conf - Este arquivo possui a configuração padrão usada pelo aplicativo do mke2fs para formatar sistemas de arquivos.
  • modules - Este arquivo possui os nomes dos módulos do kernel a serem carregados na inicialização do sistema.
  • motd - O nome deste arquivo é acrônimo para "Message Of The day", ou seja, "mensagem do dia". Ele pode ser usado pelo administrador para enviar mensagens aos usuários do sistema. O motd é um simples arquivo texto cujo conteúdo é exibido pelo processo login após um login bem sucedido, antes do shell ser executado.
  • mtab - Este arquivo possui a lista dos sistemas de arquivos que estão montados no Linux.
  • mtools - Este é o arquivo configuração do pacote mtools.
  • passwd - Este arquivo possui a lista de usuários do sistema. Existe uma entrada no arquivo para cada usuário do sistema com as seguintes informações: nome, senha (caso se use shadow password, as senhas criptografadas são armazenadas em /etc/shadow), número de UID (user identification), número de GID (group identification), informações sobre o usuário, o diretório home do usuário e o shell padrão do usuário. Por exemplo, a linha abaixo mostra a entrada criada em /etc/passwd para o usuário aluno que tem UID e GID iguais a 1001.
    aluno:x:1001:1001::/home/aluno:/bin/bash
    As informações sobre o usuário (nome, telefone, local de trabalho, etc) são incluídas com os comandos adduser e chfn e visualizadas com o comando finger.

  • printcap - Este arquivo possui a configuração das impressoras do sistema.
  • profile - Este arquivo possui a definição padrão do ambiente do sistema, pois abriga as configurações para todos os usuários. Normalmente, a configuração de aliases e funções é feita no arquivo bash.bashrc
  • profile.d - Este diretório possui as definições usadas pelos usuários.
  • protocols -Este arquivo possui a lista dos protocolos Internet DARPA que estão disponíveis para o subsistema TCP/IP.
  • rc0.d, rc1.d, rc2.d, rc3.d, rc4.d, rc5.d, rc6.d, rcS.d - Estes diretórios possuem links para os arquivos usados na inicialização do sistema. Os links dos scripts relacionados ao runlevel X estão armazenados no diretório rcX.d. Os arquivos que começam com a letra S (Start) correspondem a serviços que devem ser inicializados no runlevel correspondente, enquanto os arquivos que começam com K (Kill) indicam os scripts que devem ser executados na finalização do sistema. O último script a ser executado na inicialização é o /etc/rc.local que apenas retorna o valor zero informando que o sistema foi inicializado com sucesso. Para saber mais sobre os níveis de execução do sistema, veja init.
  • rpc - Este arquivo possui a lista dos números e respectivos nomes dos serviços RPC (Remote Procedure Calls) que são usados na implementação de serviços no modelo cliente-servidor.
  • securetty - Este arquivo possui a lista dos terminais (tty) nos quais o root pode acessar o sistema. Este arquivo é usado pelo processo login para verificar se o terminal é considerado seguro para acesso do root.
  • services - Este arquivo possui a lista dos serviços de rede e suas respectivas portas segundo recomendações da IANA (Internet Assigned Numbers Authorith).
  • shadow - Este arquivo possui as senhas criptografadas dos usuários do sistema.
  • shells - Este arquivo possui a lista dos shells que o sistema suporta.
  • shutdown.allow - Este arquivo possui a lista de usuários autorizados a reinicializar o sistema com a combinação de teclas CTRL+ALT+DEL.
  • sudoers - Este arquivo define a política de privilégios para os usuários, ou seja, como o comando sudo pode ser utilizado.

    O sudoers permite criar quatro definições: quem pode usar o privilégio definido (user_alias), a quem pode substituir (runas_alias), em quais máquinas podem executar o privilégio definido (host_alias) e quais os comandos que podem ser executados (cmnd_alias). Abaixo são mostrados alguns exemplos.

    1. Os usuários maria e pedro possuem os privilégios definidos para PROFESSORES.
      # User alias specification
      User_Alias PROFESSORES = maria, pedro
    2. Quem pode executar como CHEFE pode assumir os privilégios dos usuários root e operator.
      # Runas alias specification
      Runas_Alias CHEFE = root, operator
    3. As máquinas de LAB1 estão na rede 128.138.243.0 e as máquinas de LAB2 estão na rede 128.138.53.0.
      # Host alias specification
      Host_Alias LAB1 = 128.138.243.0
      Host_Alias LAB2 = 128.138.53.0
    4. Os comandos especificados por FIM são shutdown e reboot
      # Cmnd alias specification
      Cmnd_Alias FIM = /usr/sbin/shutdown, /usr/bin/reboot

    Para definir os privilégios, usa-se o seguinte formato:

    User Host1 = [(Runas1)] Cmnd1 : Host2 = [(Runas2)] Cmnd2 : ...
    A seguir são mostrados alguns exemplos de como definir privilégios usando as definições acima.
    1. O usuário teresa pode executar qualquer comando nas máquinas da rede 128.138.243.0.
      teresa LAB1 = ALL
    2. O usuário joao pode executar os comandos shutdown e reboot em qualquer máquina.
      joao ALL = FIM
    3. O usuário carlos pode executar nas máquinas da rede 128.138.53.0 qualquer comando do diretório /usr/local/laboratorios/ como se fosse o usuário operator.
      carlos LAB2 = (operator) /usr/local/laboratorios/
    4. Os usuários maria e pedro podem executar qualquer comando em qualquer máquina sem precisar fornecer senha.
      PROFESSORES ALL = NOPASSWD: ALL
    5. O Usuário frederico pode executar qualquer comando em qualquer máquina como se fosse o usuário operator e sem precisar fornecer senha.
      frederico ALL = (operator) NOPASSWD: ALL
    6. O usuário roberto pode executar qualquer comando como se fosse os usuários root ou operator nas máquinas da rede 128.138.243.0 e pode executar os comandos shutdown e reboot nas máquinas da rede 128.138.53.0.
      roberto LAB1 = (CHEFE) ALL : LAB2 = FIM
  • terminfo - Este diretório possui as descrições dos terminais suportados pelo sistema.
  • X11 - Este diretório possui os arquivos de configuração do sistema X.
