addgroup

addgroup [options] grupo

Descrição

Este comando adiciona um grupo ao sistema. O grupo é criado sem usuários.

Algumas opções do comando

  • --gid ID : especifica o GID do grupo sendo criado.
  • --system : especifica que o grupo a ser criado é do sistema (o valor do GID será, por padrão, menor que 1000).
  • --version : mostra informação de versão e copyright.

Exemplos

  • Para criar o grupo alunos, basta digitar:
    addgroup alunos
    O sistema então aloca um número de GID para o novo grupo e cria uma entrada em /etc/group. Quando se usa alocação dinâmica de GID, os valores ficam entre 1000 e 29999 para o grupo normal (não é grupo especial). Logo, se o grupo alunos é o primeiro a ser criado, ele terá GID igual a 1001, pois o valor 1000 já foi alocado para o grupo do usuário que instalou o sistema. Por padrão, os valores menores que 1000 de GID são reservados para o sistema (grupos especiais). Mas nada impede que o administrador aloque esses valores se eles estiverem disponíveis.
  • Para criar o grupo alunos com GID 2000, basta digitar:
    sudo addgroup --gid 2000 alunos
  • Para incluir o usuário aluno no grupo teste, onde tanto o usuário quanto o aluno já existem, digite
    sudo addgroup aluno teste

Observações

  • A configuração padrão usada pelo comando addgroup pode ser encontrada em /etc/adduser.conf.
  • O comando addgroup normalmente exige o uso do comando sudo.
  • É possível usar o comando adduser com a opção --group para criar um grupo.
  • O comando groupadd cria um novo grupo de usuários no sistema de forma idêntica ao comando addgroup.
  • Os comandos delgroup e groupdel deletam grupos do sistema.
