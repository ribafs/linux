/proc

Descrição

  • Este diretório não é um sistema real de arquivos. Na realidade, este diretório é uma interface para as estruturas de dados do kernel que oferece informações sobre os processos que estão sendo executados.
  • É importante observar que o root (administrador de sistema) não trabalha diretamente com o diretório /proc.

Exemplos de arquivos

  • asound - Este diretório possui informações sobre a placa de som da máquina.
  • bus - Este diretório possui informações sobre dispositivos como PCI e USB.

  • cmdline - Este arquivo possui informações sobre o kernel como, por exemplo, o local de armazenamento dos arquivos e as opções usadas na inicialização.

  • cpuinfo - Este arquivo possui as informações sobre o processador do sistema tais como modelo, capacidade, velocidade, fabricante, tamanho da cache, etc.
  • devices - Este arquivo possui a lista dos dispositivos E/S ativos no sistema.
  • dma - Este arquivo exibe informações sobre o DMA (Direct Memory Access). A tecnologia DMA permite que alguns dispositivos de hardware acessem a memória independentemente da CPU.
  • driver - Este diretório possui informações sobre os drivers.
  • filesystems - Este arquivo possui a lista dos sistemas de arquivos que a versão instalada do Linux suporta.
  • fs - Este diretório possui informações sobre os sistemas de arquivos.
  • interrupts - Este arquivo possui a lista das interrupções (IRQs) processadas desde a inicialização do sistema.
  • iomem - Este arquivo exibe os endereços de memória para os dispositivos de E/S.
  • ioports - Este arquivo exibe as portas dos dispositivos de E/S.
  • irq - Este diretório possui informações sobre as IRQs (Interrupt Request).
  • kcore - Este arquivo é um mapeamento da memória RAM. Normalmente, o tamanho deste arquivo representa o tamanho total da memória física. Entretanto, na arquitetuta de 64 bits, o tamanho do kcore representa a quantidade máxima de memória que a máquina pode suportar.
  • meminfo - Este arquivo apresenta informações sobre a memória do sistema.
  • modules - Este arquivo possui a lista dos módulos do kernel que estão carregados na memória.
  • mounts - Este arquivo possui a lista dos sistemas de arquivos montados (veja o comando mount). Ele normalmente possui mais informações do que o arquivo /etc/mtab como, por exemplo, as opções de montagem usadas. Embora estas informações não estejam necessariamente atualizadas.
  • net - Este diretório possui informações sobre as conexões.
  • partitions - Este arquivo exibe informações sobre as partições do sistema.
  • scsi - O arquivo scsi deste diretório possui a listagem dos equipamentos scsi atualmente em uso no sistema.
  • swaps - Este arquivo possui informações sobre a área de swap usada pelo sistema.
  • sys - Este diretório possui informações sobre os parâmetros usados pelo sistema.
  • tty -Este diretório possui informações sobre dispositivos seriais.

  • uptime - Este arquivo mostra, em segundos, o tempo que a máquina está ligada e o tempo de ociosidade do processador (ou processadores). Por exemplo, o conteúdo
    4792.58 8654.37
    indica que o computador está funcionando há mais de 1 hora e que as CPUs (são duas neste exemplo) ficaram ociosas na maior parte do tempo.
  • version - Este arquivo possui a versão, a data e a hora da compilação do kernel.
  • vmstat - Este arquivo possui informações estatísticas sobre a memória virtual usada pelo kernel. Por exemplo:
    • Número de páginas livres na memória.
      pgfree 6648052
    • Número de páginas alocadas para a tabela de páginas.
      nr_page_table_pages 1664
    • Número de páginas que foram levadas para a memória cache (pageins) e número de páginas que foram retiradas da memória cache (pageouts) desde a inicialização da máquina.
      pgpgin 455241
      pgpgout 279000
    • Número de páginas que foram levadas da área de swap do HD para a memória real (swap in) e número de páginas que foram retiradas da memória real para área de swap do HD (swap out) desde a inicialização da máquina.
      pswpin 0
      pswpout 0

Processos

É possível descobrir mais sobre os processos que estão executando no sistema analisando o /proc. Por exemplo, suponha que você está interessado no processo com PID 3966. Para descobrir informações sobre esse processo, basta analisar os arquivos do diretório /proc/3966.

  • /proc/3966/cmdline - este arquivo informa a linha de comando que inicializou o processo.
  • /proc/3966/cwd - é um link simbólico para o diretório onde o comando que iniciou o processo foi digitado.
  • /proc/3966/environ - este arquivo tem as definições das variáveis de ambiente usadas na execução do processo.
  • /proc/3966/statm - este arquivo possui informações sobre o uso de memória pelo processo.
  • /proc/3966/status - este arquivo fornece informações como nome do processo, PID, ID do processo-pai (PPID), estado do processo (Sleep ou Running), UID e GID.

Observações

  • Para obter mais informações sobre os arquivos do diretório /proc, use o comando "man proc".
