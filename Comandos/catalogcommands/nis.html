nis (network information system)

Descrição

É um serviço de disseminação de informação em rede fornecido pela SUN. Para ser mais específico, ele tem como propósito prover informações para uma rede que devem ser conhecidas por todas as máquinas da rede. Uma suposição é imaginar que suas informações de login, presentes no /etc/passwd sejam disseminadas na rede. A partir disso, será possível logar em qualquer máquina que esteja conectada à rede.

Podem existir dois tipos de servidores NIS: o Master e o Slave.

Alguns comandos do NIS

  • ypdomainname : mostra o domínio do cliente NIS.
  • ypbind : mostra o servidor de domínio NIS.
  • ypset : relaciona um cliente NIS a um servidor.
  • ypcat : mostra a base de dados do servidor NIS.
  • ypwhich : mostra o nome da máquina do servidor NIS.
  • yppoll : mostra a versão do servidor NIS.
  • ypmatch : faz uma procura na base de dados do NIS.
  • yppasswd : muda a senha do usuário na base de dados NIS.
  • ypchsh : muda o shell da base de dados NIS.
  • ypchfn : muda o nome do usuário na base de dados NIS.

Exemplo de uma estrutura nis

O servidor Master vai manter os dados da rede atualizados e repassar aos clientes ou por meio de requisições ou através do mapeamento slave. Os servidores Slave mantêm dados transferidos de um servidor Master e repassa aos clientes quando necessário. Esse último é utilizado em redes segmentadas com vários hosts ou mistas, para diminuir o tráfego em rede.

Observações e curiosidades

  • A primeira versão era conhecida também como Yellow Pages e é por isso que a maioria de seus comandos se iniciam com "yp".
  • É baseado em Chamadas de Procedimento Remoto (RPC) que utilizam um padrão de representação de dados externo.
  • Possui uma versão chamada NIS+ (NIS plus) que oferece uma camada a mais de segurança. Apesar de muito potente, é mais difícil de configurar.
