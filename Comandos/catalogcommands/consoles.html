consoles virtuais


Descrição

A partir de uma mesma máquina podemos acessar o Linux usando diferentes usuários, ao mesmo tempo e de forma independente. Para isto, basta logar cada usuário em um determinado console virtual. Um console virtual corresponde à combinação de terminal, teclado e mouse.

Os consoles virtuais são representados no Linux por arquivos especiais que ficam armazenados no /dev como tty1, tty2, tty3, etc. Na realidade, estes arquivos são interfaces para os drivers dos componentes do sistema que assim podem ser acessados usando as mesmas "system calls" dos arquivos comuns.

Quando o sistema é inicializado, normalmente são criados seis consoles de texto com o aplicativo getty e um console com interface gráfica com o sistema X. Para ver os processos que inicializa os consoles, pode-se usar o comando ps -au.

Modo texto e modo gráfico

  • Durante a inicialização, o DuZeru 12.04 cria 6 consoles no modo texto e 1 console no modo gráfico. Os consoles do modo texto são numerados de 1 a 6 (de tty1 a tty6), enquanto o console do modo gráfico é numerado como 7 (tty7).
  • O console virtual 1 (tty1) é, por padrão, utilizado para mostrar as mensagens de inicialização do sistema.
  • Estando no modo texto, para mudar de console, basta pressionar a tecla ALT junto com uma tecla de função. Por exemplo, ALT+F2 mostra o console virtual 2. No modo gráfico é preciso também pressionar a tecla CTRL junto com as duas outras teclas. Assim, podemos logar um usuário diferente (ou um mesmo usuário) em mais de um console.
  • Caso o modo de inicialização do sistema seja texto, o prompt de identificação dos usuários do sistema é apresentado no console 2. Caso o sistema seja inicializado no modo gráfico, a tela de identificação é apresentada no console 7.
